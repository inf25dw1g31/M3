### 4.3.5 DataProvider Customizado - Gerado e Corrigido por Claude.ai

**Problema:** React Admin espera campo `id`, mas API LoopBack usa `id_veiculo`, `id_cliente`, etc.

**Prompt inicial:**
> "Cria um DataProvider customizado para React Admin que:
> - Mapeie id_veiculo → id, id_cliente → id, etc.
> - Adicione campo 'id' em todos os objetos retornados
> - Mantenha o campo original (id_veiculo) também"

**Código gerado (versão 1):**
```javascript
// GERADO POR CLAUDE.AI
const normalizeData = (resource, data) => {
  const idField = {
    categorias: 'id_categoria',
    veiculos: 'id_veiculo',
    clientes: 'id_cliente',
    // ...
  }[resource] || 'id';
  
  return Array.isArray(data)
    ? data.map(item => ({ ...item, id: item[idField] }))
    : { ...data, id: data[idField] };
};
```

**Problema encontrado:** IDs compostos (cliente-favoritos) não funcionavam.

**Prompt de correção:**
> "Cliente-favoritos tem chave composta (id_cliente, id_veiculo). Como fazer o DataProvider suportar isso? Gera um ID único combinando os dois."

**Código corrigido gerado:**
```javascript
// CORRIGIDO POR CLAUDE.AI
if (resource === 'cliente-favoritos') {
  return Array.isArray(data)
    ? data.map(item => ({
        ...item,
        id: `${item.id_cliente}_${item.id_veiculo}` // ID combinado
      }))
    : { ...data, id: `${data.id_cliente}_${data.id_veiculo}` };
}
```

**Validação:** ✅ Testado com CRUD de favoritos

Claude.ai foi usado extensivamente para **debugar erros** que apareciam durante o desenvolvimento.

**Exemplos de erros resolvidos:**

#### **Erro 1: API não conecta ao MySQL no Docker**
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```

**Prompt:**
> "Erro: API não conecta ao MySQL. Estou usando docker-compose. A API tem DB_HOST=localhost mas não funciona."

**Solução:** ✅ Mudou `DB_HOST: localhost` → `DB_HOST: db`

#### **Erro 2: React Admin não mostra dados**
```
GET http://localhost:3000/veiculos

**Prompt:**

**Resposta de Claude.ai:**

#### **Erro 3: Triggers MySQL não funcionam**
```
Error: Trigger 'manutencao_criada_muda_estado' has wrong syntax

**Prompt:**
> "Trigger MySQL dá erro de sintaxe. Código: CREATE TRIGGER manutencao_criada_muda_estado..."

> "Usa `DELIMITER $$` antes do trigger e restaura depois. O problema é que o `;` dentro do trigger termina o comando antes do END."

**Solução:** ✅ Adicionou DELIMITER conforme sugerido
